System Architecture

Understanding the design and structure of the Bookstore API

Architecture Overview

The Online Bookstore API follows a clean, layered architecture pattern that promotes separation of concerns, testability, and maintainability.

Design Philosophy

Our architecture is built on SOLID principles, ensuring that each layer has a single responsibility and can be tested independently.

Layered Architecture

The system is organized into four distinct layers, each with specific responsibilities:

Presentation Layer

Handles HTTP requests and responses, routing, and middleware.

  • Controllers
  • Middleware
  • Swagger/OpenAPI
  • Error Handling

Application Layer

Contains business logic, commands, queries, and validators.

  • MediatR Commands
  • MediatR Queries
  • Validators
  • DTOs

Infrastructure Layer

Implements data access, caching, and external services.

  • Entity Framework Core
  • Repositories
  • Redis Caching
  • Authentication

Domain Layer

Contains core business entities and domain logic.

  • Entities
  • Value Objects
  • Domain Events
  • Business Rules

Layer Dependencies

Presentation Layer
        ↓
Application Layer
        ↓
Infrastructure Layer ← Domain Layer
        ↓
Database / External Services

Design Patterns

CQRS (Command Query Responsibility Segregation)

Separates read and write operations for better scalability and performance.

  • Commands: Operations that modify state (Create, Update, Delete)
  • Queries: Operations that retrieve data without side effects
  • Handlers: Process commands and queries using MediatR

Repository Pattern

Abstracts data access logic and provides a collection-like interface.

  • Generic repository for common operations
  • Specialized repositories for complex queries
  • Unit of Work for transaction management

Dependency Injection

Loose coupling through constructor injection and service registration.

  • Built-in .NET Core DI container
  • Extension methods for service registration
  • Scoped, transient, and singleton lifetimes

Middleware Pipeline

Request processing through a chain of middleware components.

  • Authentication & Authorization
  • Request/Response Logging
  • Exception Handling
  • Performance Monitoring
